## 1. Overview

To allow volunteer staff at Peninsula Tamil School to update the website content (such as announcements, class offerings, schedule adjustments, event details, and photos) without editing code or requiring code deploys, the application integrates with **Google Sheets**. 

The app reads public CSV streams generated by published Google Sheets. If the network is unavailable, if a sheet is misconfigured, or if formatting is corrupt, the app seamlessly falls back to static content stored locally inside `src/data/content.js`.

---

## 2. How it Works

1. **Content Source**: School coordinators update content in a Google Spreadsheet.
2. **Web Stream**: The spreadsheet (or specific tabs within it) is published to the web as a CSV file.
3. **App Integration**: When a user visits the website, the React application makes asynchronous `fetch` requests to these CSV URLs.
4. **Parser**: A custom lightweight, zero-dependency CSV parser (`parseCSV` and `csvToObjects` in `src/services/cmsService.js`) processes the raw text.
5. **Dynamic Updates**: The parsed rows are converted to JavaScript objects, validated, and rendered dynamically.

## 3. Environment Variables

The CMS endpoints are configured in the frontend using environment variables. To configure these, create or update `.env` (or `.env.local` for local development) at the project root with the following keys:

```env
# Google Sheets Published CSV URLs
VITE_CMS_SETTINGS_URL="https://docs.google.com/spreadsheets/d/[SHEET_ID]/export?format=csv&gid=[SETTINGS_TAB_ID]"
VITE_CMS_ANNOUNCEMENT_URL="https://docs.google.com/spreadsheets/d/[SHEET_ID]/export?format=csv&gid=[ANNOUNCEMENT_TAB_ID]"
VITE_CMS_EVENTS_URL="https://docs.google.com/spreadsheets/d/[SHEET_ID]/export?format=csv&gid=[EVENTS_ENG_TAB_ID]"
VITE_CMS_EVENTS_TAMIL_URL="https://docs.google.com/spreadsheets/d/[SHEET_ID]/export?format=csv&gid=[EVENTS_TAM_TAB_ID]"
VITE_CMS_CLASSES_URL="https://docs.google.com/spreadsheets/d/[SHEET_ID]/export?format=csv&gid=[CLASSES_TAB_ID]"
VITE_CMS_GALLERY_URL="https://docs.google.com/spreadsheets/d/[SHEET_ID]/export?format=csv&gid=[GALLERY_TAB_ID]"
```

*Replace `[SHEET_ID]` with your specific Google Spreadsheet ID, and `[TAB_ID]` with the specific sheet's GID.*

## 4. Setting Up a Google Sheet as a CMS

To connect a new or existing Google Sheet tab to the website:

1. Open your Google Spreadsheet.
2. Click **File** > **Share** > **Publish to web**.
3. Under the **Link** tab, select the specific sheet tab (e.g., "Events") and set the format to **Comma-separated values (.csv)**.
4. Click **Publish** and confirm.
5. Copy the generated link. It will look like:
   `https://docs.google.com/spreadsheets/d/[SHEET_ID]/pub?gid=[TAB_ID]&output=csv`
6. Alternatively, use the cleaner export format URL in your `.env` file:
   `https://docs.google.com/spreadsheets/d/[SHEET_ID]/export?format=csv&gid=[TAB_ID]`

## 5. Data Schemas & Field Requirements

To ensure the custom parser successfully extracts and maps data, you **MUST** follow the column header structures detailed below. Columns are case-insensitive and whitespace-trimmed.

### General School Settings (`VITE_CMS_SETTINGS_URL`)
This sheet handles key-value pairs representing global school variables.

* **Headers**: `key`, `value`

| Key | Description |
|---|---|
| `enrollmentLink` | URL to registration form (e.g., Google Form) |
| `email` | School email address |
| `phone` | School phone number |
| `addressVenue` | Name of school venue (e.g., Hillsdale High School) |
| `addressFull` | Full school address |
| `scheduleDay` | Day of the week (e.g., Sunday) |
| `scheduleTime` | Class hours (e.g., 10:20 AM - 12:20 PM) |
| `facebook` | Facebook page URL |
| `instagram` | Instagram page URL |
| `nonprofitEin` | 501(c)(3) EIN number |
| `nonprofitStatus` | Nonprofit text (e.g., 501(c)(3) Nonprofit Organization) |
| `heroTitleEn` | English Homepage Hero Title |
| `heroTitleTa` | Tamil Homepage Hero Title |
| `heroSubtitleEn` | English Homepage Hero Subtitle |
| `heroSubtitleTa` | Tamil Homepage Hero Subtitle |

**Example Sheet:**
| key | value |
|---|---|
| enrollmentLink | https://forms.google.com/your-form |
| email | info@peninsulatamilschool.org |
| phone | 408-203-6500 |
| heroTitleEn | Empowering Tamil Heritage |
| heroTitleTa | தமிழ் பாரம்பரியத்தை மேம்படுத்துதல் |

### Announcement/Alert Banner (`VITE_CMS_ANNOUNCEMENT_URL`)
Manages the banner displaying crucial announcements at the top of the page (e.g., "Registration Open").

* **Headers**: `active`, `text_en`, `text_ta`, `link_url`

| Column Name | Description |
|---|---|
| `active` | `TRUE` to display the banner, `FALSE` to hide it. |
| `text_en` | English text for the banner. |
| `text_ta` | Tamil text for the banner (if left blank, defaults to `text_en`). |
| `link_url` | (Optional) URL to navigate to when the banner is clicked. |

**Example Sheet:**
| active | text_en | text_ta | link_url |
|---|---|---|---|
| TRUE | Registration for 2026-2027 academic year is now open! | 2026-2027 கல்வியாண்டிற்கான சேர்க்கை இப்போது தொடங்கப்பட்டுள்ளது! | https://forms.google.com/your-form |

### Events Calendar (`VITE_CMS_EVENTS_URL` & `VITE_CMS_EVENTS_TAMIL_URL`)
The Events page displays a rich calendar. Because we support bilingual layouts, you configure an English events sheet and a Tamil events sheet.

* **Headers**: `month`, `day`, `title`, `time`, `location`, `description`, `featured`

| Column Name | Description |
|---|---|
| `month` | Abbreviated month name (e.g., "OCT" or "அக்டோபர்"). |
| `day` | Calendar day string (e.g., "15" or "15-18"). |
| `title` | Title of the event (**Required**). |
| `time` | Time of event (e.g., "10:20 AM - 12:20 PM"). |
| `location` | Venue of event. |
| `description` | Descriptive text of the event. |
| `featured` | Set to `TRUE` to pin to the home page "Upcoming Events" section, `FALSE` otherwise. |

**Example Sheet (English):**
| month | day | title | time | location | description | featured |
|---|---|---|---|---|---|---|
| SEP | 07 | First Day of Classes | 10:20 AM | Hillsdale High School | Welcome assembly & curriculum overview | TRUE |
| OCT | 19 | Pongal Celebration | 10:00 AM | School Grounds | Traditional food, music, and games | TRUE |

Populates the interactive photo gallery page.

## 6. Fallback Mechanism

Robust software design requires the system to handle failures gracefully. The application implements this via standard fallback boundaries:

1. **Timeout Handling**: Requests to Google Sheets have a strict timeout (e.g., 5 seconds for events, 4 seconds for announcements). If the sheet takes too long to load, it will time out and invoke fallback code immediately.
2. **Structure Verification**: After downloading, the parser validates that the CSV headers and data conform to structural requirements. If columns are missing or formatted incorrectly, the CMS loader catches the error and switches to fallback content.
3. **Local Standby Data**: Fallback content is sourced from `src/data/content.js`. Developers should keep `content.js` relatively up-to-date with current content so that the site remains operational and complete even in complete offline scenarios.

## 7. Troubleshooting

* **Problem: A content update in Google Sheets isn't showing up on the live site.**
  1. *Check browser cache*: Open the site in Incognito Mode.
  2. *Verify web publication*: Paste the environment variable URL into a browser tab and verify it triggers a download of the exact CSV you expected.
  3. *Header naming*: Confirm that the header row in your Google Sheets tab perfectly matches the required names (e.g., `text_en` instead of `text-en`).
  4. *Developer Tools*: Check the browser console. The CMS fetching code logs warnings when it fails, which will point to the exact cause (e.g., "CORS block", "bad column headers", or "Timeout").

* **Problem: The Top Banner is missing.**
  * Ensure the first data row under the header in the Announcement tab has the `active` column set to exactly `TRUE`. If it's empty, `FALSE`, or misspelled, the banner remains hidden.
